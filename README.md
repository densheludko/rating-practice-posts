## Проект: Тематические посты

### Необходимо создать сервис на python, который позволяет опубликовывать небольшие тематические посты.В этих постах могут (и должны) содержаться фотографии и изображения. 

### Структура таблиц (доменные сущности).

### Таблица постов (posts).:
- ### id (int/guid): первичный ключ
- ### text (text/string): текст сообщения. Макс длинна - 200 символов. 
- ### date (datetime): дата создания. дата создания. 

### Таблица прикрепленных файлов (post_files):
- ### id (int/guid): первичный ключ
- ### post_id (int/guid): id поста.
- ### file_id (guid): id файла. 
- ### created_at: дата создания (datetime). дата создания. 
- ### labels (array/json): Найденные классы/объекты. Список тем нужно взять из классификатора нейронной сети (несколько штук на выбор). 

### Таблица файлов (files):
- ### id (guid): первичный ключ
- ### name (text/string): имя файла.. 
- ### path (text/string): физичесеий путь на диске. 
- ### created_at (datetime): дата создания. дата создания. 

### Структура api.

#### Публикация поста (POST /api/post).На вход передавать json вида: ```{text: "текст сообщения", files: [{"name": "имя файла", data: "base64 закодированный файл"}]} ```. Необходимые действия:
- #### Необходимо, что бы длина текста укладывалась от 1 дл 200 симоволов. Пустой текст нельзя сохранять. 
- #### Загружать можно только файлы изображения (jpeg,jpg,png). После успешной валидации необходимо
    - ### Сохранить файл на диске (создать папку для файлов)
    - ### Подключить в качестве классификатора изображений нейронную сеть (сверточную, или трансформер). В данном случае предлагаю вариант yolo.
    - ### Получить список найденных объектов на картинке.
    - ### Создать запись в таблице files (id, имя файла, физический путь на диске, время создания, темы/объекты/классы (labels))
    - ### Сохранить пост в таблице posts (id, текст, дата).
- #### После успешного сохранения вернуть json ```{"result": "id поста"} ```, иначе: ```{"error": "текст ошибки"} ```

#### Получение постов (GET /api/post). На вход принимать json  Необходимо вернуть массив постов в виде json ```{[{"id": "id поста", text: "текст поста", created_at: "дата создания", files: [{"id": "id файла", "name": "имя файла", "labels":"имена найденных объектов"}]}]} ```

#### Удаление поста (DELETE /api/post/{id}). Необходимо удалить пост и все его записи в дополнительных таблицах (и файлы)

#### Загрузка файла (GET /api/files/{id}). Необходимо отдать файл изображения.

### Работа с классификатором изображения (YOLO). Для простоты я предлагаю использовать стандартную модель YOLO. 

#### Пример использования (просмотр изображения + метки): 
``` 

    from ultralytics import YOLO

    model = YOLO("yolov8m.pt")
    results = model("1.jpg")
    results[0].show()

 ```
 ### Пример, где можно получить список классов/меток (наименований): 
 ``` 

    from ultralytics import YOLO

    model = YOLO("yolov8m.pt")
    results = model("1.jpg")
    names = results[0].names #наименование всех классо
    cls = results[0].boxes.cls #массив индексов, какие классы были найдены
    conf = results[0].boxes.conf #массив индексов, какие классы с какой вероятностью были найдены.

    name = names[cls[0]] # так можно получить имя найденного класса
    second_name = names[cls[1]] # так можно получить имя найденного класса

 ```
